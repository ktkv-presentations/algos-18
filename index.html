Рекурсия

Рекурсия

метод решения задачи, при котором функция вызывает саму себя для решения подзадачи

Особенность рекурсии

наличие одного или нескольких условий выхода (базовых случаев), которые останавливают дальнейшие рекурсивные вызовы.

Принцип работы рекурсивных функций

Базовый случай

(условие выхода)

условие, при котором рекурсия прекращается

Рекурсивный случай

часть, где функция вызывает саму себя с изменёнными параметрами для приближения к базовому случаю

Пример базового рекурсивного алгоритма


def factorial(n):
    if n == 1:
        return 1
    else:
        return n * factorial(n - 1)

Области применения рекурсии

Сортировка данных


def quicksort(arr):
    if len(arr) <= 1:
        return arr
    else:
        pivot = arr[0]
        less = [x for x in arr[1:] if x <= pivot]
        greater = [x for x in arr[1:] if x > pivot]
        return quicksort(less) + [pivot] + quicksort(greater)  # рекурсивный случай

Области применения рекурсии

Обработка рекурсивных математических структур


def fibonacci(n):
    if n == 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n - 1) + fibonacci(n - 2)

Преимущества рекурсии

  • Упрощает решение сложных задач, позволяя их выразить в краткой и понятной форме
  • Подходит для задач, где данные или структура задачи имеют иерархическую или повторяющуюся природу

Недостатки рекурсии

  • Может привести к переполнению стека (Stack Overflow) при больших глубинах рекурсии
  • В некоторых случаях рекурсивные алгоритмы менее эффективны по времени и памяти, чем итерационные решения
